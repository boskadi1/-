# v1.51 卸货流程修复

## 修复原因

v1.50 在识别到卸货区入口后仍保持 `10°/10°` 寻线视角。只有黑色圆块先在该视角进入 READY 并稳定 300 ms，摄像头才会转到 `10°/30°`。实际场地中圆块可能无法从寻线视角通过筛选，导致小车无法进入后续对齐和卸货。

## v1.51 流程

```text
识别卸货区宽横带
  → 停止循迹
  → 摄像头立即从 10°/10° 转到 10°/30°
  → 等待云台到位
  → 左70、右60低速搜索圆形落货点
  → 横向偏差：低速原地左右转
  → 圆形偏远：低速前进
  → 圆形过近：低速后退
  → 圆形稳定对准300 ms
  → 执行卸货
```

如果8秒内仍未识别到圆形，小车停车，不会无限向前行驶。

## 使用文件

```text
openmv/OpenMV_sensor_load_exit_unload_circle_v2.4.py
firmware/stm_unload_zone_recognition_v151.elf
stm32/
```

OpenMV v2.4 与 v1.51 STM32 固件配套使用。
