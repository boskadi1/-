# v1.55 循迹PID与电机编码器闭环

本版本仅保存在本地，不上传 GitHub。

## 两层闭环

1. 外环循迹PID
   - P：横向偏差和线路方向产生基础差速。
   - I：持续偏向同一侧时逐步增加修正，最大为5。
   - D：保留原来的滤波微分项，抑制快速偏移。
2. 内环轮速PID
   - 左右轮分别比较目标速度和编码器反馈速度。
   - 实际速度低于目标时增加该侧PWM。
   - 实际速度高于目标时降低该侧PWM。
   - 左右轮PID互相独立。
   - 编码器没有有效脉冲时自动退回原来的开环PWM斜坡，避免断线后PWM一直增加到100。

## 编码器接线

工程原来已经把PF0～PF7配置为上升沿输入，但没有处理中断。v1.55采用：

```text
PF0/PF1  左前轮或唯一左轮编码器 A/B
PF2/PF3  左后轮编码器 A/B（两轮车可不接）
PF4/PF5  右前轮或唯一右轮编码器 A/B
PF6/PF7  右后轮编码器 A/B（两轮车可不接）
```

如果每侧只有一个编码器，只连接“唯一左轮”和“唯一右轮”两组即可。

## 蓝牙数据

每500 ms返回：

```text
TEL,E=偏差,C=循迹修正,L=目标/反馈/PWM,R=目标/反馈/PWM,N=左右原始脉冲,V=有效标志
```

示例：

```text
TEL,E=8,C=9,L=94/86/98,R=61/64/60,N=86/64,V=111
```

- `L=94/86/98`：左轮目标94、反馈86、实际PWM98。
- `R=61/64/60`：右轮目标61、反馈64、实际PWM60。
- `N=86/64`：本次50 ms采样得到的左右原始编码器脉冲数。
- `V=111`：左编码器有效、右编码器有效、循迹数据有效。

## 必须标定

`Core/Inc/car_config.h`：

```c
#define ENCODER_COUNTS_PER_SAMPLE_AT_FULL_SPEED 100U
```

当前采样周期是50 ms。先把小车架空并观察蓝牙反馈。如果PWM=100时反馈不是100，应按下面方式修改：

```text
新参数 = PWM=100时蓝牙原始50ms脉冲数
```

主要PID参数：

```c
MOTOR_SPEED_PID_KP
MOTOR_SPEED_PID_KI
MOTOR_SPEED_PID_KD
MOTOR_SPEED_PID_CORRECTION_MAX
CAR_LINE_KI_NUM
CAR_LINE_KI_DEN
```
